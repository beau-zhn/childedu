WELCOME TO MY USER GUIDE

A full user guide with all options described presented as a booklet

  1. Registering onto your website

For making purchases a user should have an account and for this user must complete the registration form. Data that should be entered is the username and the password. The password must have more than 8 values.

In case everything will be correct then a message about completed registration will appear.

Picture 53. The registration page.

  1. Logging in website

In case the user does not have an account he can press the button that will redirect to the registration page. But if the user has an account he must enter the form, its username and password which should match the information in the database. In case the entered info will be correct, the user will log in to a website and be allowed to make purchases.

Picture 54. The login page.

  1. Adding courses to the basket

After choosing the appropriate category in which the user is interested in the courses will be displayed and right after the description of the course there is a button “WANT THIS COURSE”, by clicking on this button adds the product to the basket is happens. This could be seen on the top of the page where the shopping cart is located, Chosen courses will be displayed, and the final cost will be displayed there too.

Picture 55. The courses and the shopping cart page.

  1. Filling the checkout

On this page, the user should fill the checkout form. All in all, there is 8 field that should be filled. They are the first and last name of the child, current address, education place, parent name, age, number, and email. Here are some special points that should be mentioned. Firstly filling all of them is obligatory, leaving one empty will not allow continuing further.
Secondly, age and the number should consist only of numbers.
Thirdly, age can not be negative, moreover, its value should range from 0 to 150 maximum.

Fourthly, the number should be entered as in an example provided on the website which is 8-xxx-xxx-xx-xx.

Fifthly, email should have an appropriate symbol which is @.

Picture 56. The checkout page.

  1. Log out from the website

In case the user ends his process of buying courses or just want to stop the current session user can log out. This is possible on any page. On the top of the website, there will be a message that displays the username that is currently logged in and the button to log out. By clicking on this button user will end his session and can pass to the unlogged version of the website.

Picture 57. The main page with the logged-in user.

  1. Backup the site from Beget

In the beget the section “Backup” exists. This section is used for creating the ‘copy of the site’ and making it possible to recover the data.

Picture 60. The backup section in the Beget.

  1. Recovery from Beget

In the beget is two options of recovering are possible. Recovering the file or the whole database. In the first variant, you can choose the specific file that should be recovered and click the blue button. Databases are recovered in an alike way. Go to the tab Databases after that you should select a date and then press the blue button.

Glossary 

Account – is the user’s personal identifier that helps to receive certain access for functions of the website.

Button - control element that does specific operation when you click on it.

Backup - copying the existing data for further usage in case of loss or damaging the data.

Database - is the set of organized data.

Form - is a window where needed data for collection is displayed and filled.

Phpmyadmin - - free software tool where you can manage the databases.

Password - is the set of letters, symbols and numbers that will help to allow to enter to the system.

Redirect - is the process of transferring the user from one page to other.

Recovery - replacing damaged or lost data with one which was saved previously.

Username - is the name that the user will set to his account and will help login into the system.
